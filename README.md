Московский зоопарк — консольное приложение учёта

Описание

Приложение позволяет вести учёт животных и инвентаря Московского зоопарка.  
Предоставляет интерфейс для добавления животных и вещей, отображения отчётов, учёта потребления еды и формирования списка животных для контактного зоопарка.

Проект построен с использованием принципов ООП, SOLID, применяет внедрение зависимостей (DI) и покрыт юнит-тестами.

---

>>> Функциональность

- Добавление животных разных видов: Обезьяна, Кролик, Тигр, Волк  
- Добавление вещей в инвентарь: Стол, Компьютер  
- Проверка здоровья животного перед добавлением (через ветеринарную клинику)  
- Расчёт общего потребления еды всеми животными  
- Отображение количества животных  
- Формирование списка животных для контактного зоопарка  
- Отображение инвентаризационных номеров животных и вещей

>>> Применение SOLID

Single Responsibility (S) | Каждый класс выполняет одну задачу (например, VeterinaryClinic проверяет только здоровье животных). |
Open/Closed (O)           | Добавление новых животных и вещей без изменения существующих классов.                                 |
Liskov Substitution (L)   | Все животные наследуются от общего интерфейса и могут заменять друг друга.                            |
Interface Segregation (I) | Минимальные специализированные интерфейсы (например, Herbo).                                        |
Dependency Inversion (D)  | Внедрение зависимостей через Spring; ZooService зависит от абстракций, а не конкретных реализаций.  |
