Модуль "Учет финансов" - Отчет
Общая идея решения
Реализован полнофункциональный модуль финансового учета с поддержкой 8 паттернов проектирования. Основной функционал:

✅ Управление банковскими счетами (создание, пополнение, списание)

✅ Система категорий доходов/расходов

✅ Финансовые операции с валидацией

✅ Наблюдатель за изменениями баланса

✅ Экспорт данных в JSON

✅ DI контейнер для управления зависимостями

Реализованные принципы SOLID
SRP (Single Responsibility)
BankAccountFacade - только работа со счетами

DomainFactory - только создание объектов

LowBalanceObserver - только отслеживание низкого баланса

OCP (Open-Closed)
DataImporter - расширяем для новых форматов

DataVisitor - добавляем новые форматы экспорта

BalanceObserver - добавляем новых наблюдателей

LSP (Liskov Substitution)
ObservableBankAccount полностью заменяет BankAccount

Все репозитории взаимозаменяемы

ISP (Interface Segregation)
Repository - мелкие специализированные методы

Command - минимальный интерфейс

BalanceObserver - единственный метод

DIP (Dependency Inversion)
Фасады зависят от абстракций репозиториев

DI контейнер инжектирует зависимости

Реализованные принципы GRASP
High Cohesion
Каждый класс имеет четкую зону ответственности:

Фасады группируют связанную функциональность

Наблюдатели отвечают только за уведомления

Команды инкапсулируют сценарии

Low Coupling
Зависимости через конструкторы

Интерфейсы вместо конкретных реализаций

События вместо прямых вызовов

Реализованные паттерны GoF (8 шт.)
Фасад - BankAccountFacade - упрощает работу со счетами

Команда - Command и реализации - инкапсуляция сценариев

Декоратор - TimingDecorator - добавление тайминга без изменения кода команд

Шаблонный метод - DataImporter - общий алгоритм импорта

Посетитель - DataVisitor и JsonExportVisitor - обработка разных типов данных

Фабрика - DomainFactory - централизованное создание объектов с валидацией

Прокси - BankAccountRepository - кэширование и контроль доступа

Наблюдатель - BalanceObserver и LowBalanceObserver - реакция на изменения баланса

DI контейнер
Реализован контейнер DIContainer, который:

Управляет жизненным циклом объектов

Автоматически разрешает зависимости

Обеспечивает тестируемость и гибкость

Инструкция по запуску
Требования
Java 11 или выше

Gradle 7.0 или выше

Сборка и запуск
Сборка проекта:

bash
./gradlew build
Запуск приложения:

bash
./gradlew run
Создание JAR:

bash
./gradlew jar
Запуск из JAR:

bash
java -jar build/libs/hse-bank-finance-tracker-1.0.0.jar
Структура проекта
Проект использует стандартную структуру Gradle. Исходный код в src/main/java, тесты в src/test/java.
